# docs/handoffs/ — dated session handoffs & deliverables

Dated, point-in-time documents — the opposite of `docs/status/` (which is a single file updated in place).

- The `/handoff` prompt writes here: `YYYY-MM-DD-<slug>-handoff.md`.
- Plans, reports, analyses, audits also belong here (or `docs/plans/`), named `YYYY-MM-DD-<slug>.md`.
- These are committed repo content → **no secrets**. Reference artifacts by path; redact any credential.
- Pair with a living `docs/status/<slug>.md`: the handoff is the session-end snapshot, the status file is the running state.
